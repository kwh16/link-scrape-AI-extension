# 要件定義書: YouTube History AI Filter Extension

## 1. プロジェクト概要
YouTubeの再生履歴ページから動画データをスクレイピングし、ユーザーがチャットUI上で指定した条件（特定のチャンネル、特定の内容など）に基づいて、Gemini APIを用いて該当する動画のリンク一覧を抽出・提示するChrome拡張機能。

## 2. 技術スタック
* **フレームワーク:** React + TypeScript (Vite環境)
* **スタイリング:** Tailwind CSS
* **拡張機能規格:** Manifest V3
* **LLM API:** Gemini API (モデル: Gemini 1.5 Flash または指定のFlash Previewモデル)
* **APIキー管理:** Viteの環境変数 (`.env`) に記述し、ビルド時に埋め込む（個人利用前提のため）

## 3. システムアーキテクチャと役割
* **Popup (React):** ユーザーとのチャットインターフェースを提供する。ユーザーの入力を受け取り、Gemini APIとの通信を行い、結果を表示する。
* **Content Script:** YouTubeの再生履歴ページ (`https://www.youtube.com/feed/history`) にインジェクトされ、DOMの操作（スクロール）とデータのスクレイピングを行う。
* **Background (Service Worker):** （必要に応じて）PopupとContent Script間のメッセージングの仲介や、拡張機能のライフサイクル管理を行う。

## 4. 処理のフロー
本拡張機能は、以下の対話および実行フェーズに沿って動作する。

### フェーズ1: 対話と意図確認 (Popup UI)
1. ユーザーがChromeの拡張機能アイコンをクリックし、Popupを開く。
2. システムプロンプトにより、Geminiから「YouTubeの履歴から何を抽出しますか？」という初期メッセージを表示する。
3. ユーザーが抽出条件（例: 「〇〇チャンネルの動画」「〇〇に関する動画」など）をテキストで入力する。
4. Geminiがユーザーの条件を解釈し、「〇〇に関する動画ですね。スクレイピングを実行してよろしいですか？」と実行の確認（意図確認）をチャット上に返す。

### フェーズ2: スクレイピング実行 (Content Script)
1. ユーザーが「はい」等の同意を示した場合、PopupからContent Scriptへスクレイピング実行のメッセージを送信する。
2. Content Scriptは対象ページ (`https://www.youtube.com/feed/history`) において、**自動で一番下までスクロールする処理を10回実行**し、遅延読み込み(Lazy Load)されている履歴データをDOM上に展開する。
3. スクロール完了後、画面上の動画要素から以下の情報を取得し、JSON配列の形式で構築する。
   * 動画タイトル (`title`)
   * 動画URL (`url`)
   * チャンネル名 (`channel`)
   * ※データ構造例: `[{ "title": "...", "url": "...", "channel": "..." }]`
4. 構築したJSON配列をPopup側に返却する。

### フェーズ3: AIによる抽出と結果出力 (Gemini API)
1. Popupは、Content Scriptから受け取った「スクレイピングデータのJSON配列」と「ユーザーが最初に指定した条件」を結合し、Gemini APIへ送信する。
2. GeminiはJSONデータを解析し、条件に合致する動画データのみを抽出する。
3. 抽出された動画のタイトルとURLのリストを、Popupのチャット画面にテキスト（マークダウンのリスト形式など）で出力する。

## 5. UI/UXの要件
* **チャット形式:** ユーザーの発言とAIの発言が視覚的に区別できるモダンなチャットUI。
* **ローディング表示:** スクロール処理中やGemini APIのレスポンス待ちの間は、進行中であることがわかるインジケーター（スピナーや「処理中...」のテキスト）を表示する。
* **エラーハンドリング:** YouTubeの履歴ページ以外で実行された場合、「YouTubeの履歴ページで実行してください」という警告をチャット上に表示する。
